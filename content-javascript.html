Testing and Automation

At Xerox I wrote test scripts in JavaScript that would run functional tests on the circuit board sub-components of their printer products. As the circuit boards would reach the end of the assembly line, prior to final assembly, they would be put into a test fixture. A bed of needles would be lowered onto the circuit board and connect it a computer on the test fixture. I designed the interface hardware, programmed the embedded microcontrollers in C, and wrote the functional testing software on the computer in JavaScript. The software would put the circuit board through its paces and catalog any unexpected behavior. Tests were prioritized and results uploaded to a central server for data mining. These fixtures were essential for Xerox's six-sigma management strategy and warranty insurance research.


My position at Integra was as a software tester. About half my job was manual testing and the other half I spent on maintaining their suite of automated testing scripts, written in JavaScript. During my time there, I added dozens of test cases, made several improvements to the utility libraries, and assisted in the design and build out of a Build Verification Test (BVT) program that automatically validated the nightly build.

Back-End: Node.js and Embedded Web Server

I became interested in Node (or Node.js) for it's ability to execute Back-end/Server-side functions with programs written in JavaScript. This is handy in many places, but my personal interest was in open-source, embedded web servers such as the Raspberry Pi. The examples below were originally designed and run on a Raspberry Pi. For purposes of demonstration on this site, I ported the code to Node servers running Windows 2008 Server in the Amazon EC2 Cloud. The code examples below discuss the backend components of the examples listed above in the Front-End section.

The code examples never progressed past the prototyping stage, so they used simple CSV files for data storage. For scalability, a SQL database would normally be used, but CSV is great for small websites and quick prototyping. I am currently taking courses at Code School to practice building and maintaining SQL databses. Node.js has several excellent SQL API libraries.


  • SPOT Tracker Map - The server linked to my SPOT satellite tracker data is published as an XML file. Only the last 50 data points are available. As new data comes in, old data is discarded, but I never use more than 50 data points in a day. I wrote an application in Node that downloads the XML file and parses the data into a CSV file. This data is then retrieved by the client-side JavaScript running in the Client's browser.
